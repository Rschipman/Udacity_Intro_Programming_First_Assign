What's the Web Made of?


The web is a collection of documents written in html along with other resources such as images and videos, as well as programs and services that interact with the documents and resurces.


Vocabulary

  • HTML, Hypertext Markup Language, is a language used to structure and format webpages.
  • Hypertext is a form of text in which documents can refer (link) to other documents and resources.
  • Markup is the text surrounding our plain text in HTML used to format the HTML based webpage.
  • HTTP, Hypertext Transfer Protocol, is the name of the rules the browser is given to render our webpage.
  • Syntax is another name for the rules of a language's grammar. Just like any other language computer languages have syntax rules to follow.
  • Whitespace includes spaces, tabs, and line breaks. When the browser renders an HTML file it treats a run of whitespace as a single space character. In order to create a line break that will show up in the browser used the br tag.

Different types of elements

  1. Block Elements Block Elements are rendered in the browser as boxes or blocks surrouding our text with a determined height, width, and margin. Block Elements can contain inline elements, but inline elements cannot contain block elements
    Examples of Block Elements are
    • p (paragraph)
    • ul (unordered list)
    • ol (ordered list)
    • div (document division)
    • li (list item)
    • h1 through h6 (heading and subheadings
  3. Inline Elements Inline Elements only occupy the space bound by the tags defining the element.
    Exampes of Inline Elements are
    • em (emphasises with the default being italics)
    • strong (bold text)
    • sup (Superscript)
    • sub (Subscript)
    • mark (highlight text default: yellow)
  5. Void Elements Void elements only have one opening tag and no closing tag, because unlike implied close tag elements elements like br have no contents.

Implied Close Tags

Some tags like p and br have implied close tags and do not need a closing tag when written in HTML. The browser will know the previous element is done when it encounters the next element.
